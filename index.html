Price Tracker Application

A comprehensive guide to the React & Firebase Price Tracker project.

View Live Application

Technology Stack

This application is built with modern, scalable technologies to provide a seamless and real-time user experience.

Frontend

  • React: A JavaScript library for building dynamic user interfaces.
  • Tailwind CSS: A utility-first CSS framework for rapid and responsive styling.
  • Firebase SDK (v9): For seamless frontend-backend communication.

Backend & Database

  • Firebase: A comprehensive platform for backend services.
  • Cloud Firestore: A NoSQL, real-time database for storing product data.
  • Cloud Functions for Firebase: Serverless backend running Node.js for web scraping.

Web Scraping & Infrastructure

  • Axios & Cheerio: For fetching and parsing HTML content from product pages.
  • ScraperAPI: A proxy service to prevent scraping requests from being blocked.
  • Firebase Hosting: For deploying and serving the React frontend.

Application Workflow

The application follows a modern serverless architecture. The frontend React app handles the UI, while a Firebase Cloud Function manages the web scraping, ensuring security and scalability.

1. User Action
User pastes a product URL (Amazon AU) and clicks "Track Price".
↓
2. Frontend Request
React app calls the scrapeUrl Firebase Cloud Function, sending the URL.
↓
3. Backend Scraping
The Cloud Function uses a proxy (ScraperAPI) to fetch the HTML from the product page.
↓
4. Data Extraction
Cheerio parses the HTML, extracting the title, price, and image URL.
↓
5. Data Response
The function returns the extracted data as a JSON object to the React app.
↓
6. Database Storage & UI Update
React saves the new product data to Cloud Firestore, and the UI updates in real-time.

Setup & Deployment

Follow these steps to set up and deploy your own instance of the application.

1. Firebase Project Setup

  1. Go to the Firebase Console and create a new project.
  2. Upgrade your project to the "Blaze (Pay as you go)" plan. This is required for Cloud Functions to make external network requests.
  3. In the Firebase dashboard, enable Authentication (Anonymous provider), Firestore Database (Production mode), and Functions.
  4. Register a new Web App in your project settings and copy the `firebaseConfig` object.

2. ScraperAPI Setup

  1. Go to scraperapi.com and sign up for a free account.
  2. From your dashboard, copy your API Key.

3. Local Development

  1. Clone the project repository.
  2. Install the Firebase CLI globally: 
    npm install -g firebase-tools
  3. Log in to Firebase: 
    firebase login
  4. Initialize the project in the root folder: 
    firebase init
  5. Backend: Navigate to `functions`, run `npm install`, and paste your ScraperAPI key into `index.js`.
  6. Frontend: Navigate to `client`, run `npm install`, and paste your `firebaseConfig` object into `src/App.js`.
  7. Start the app from the `client` folder: 
    npm start

4. Deployment

  1. From the `client` folder, create a production build: 
    npm run build
  2. Navigate back to the project's **root** folder.
  3. Run the deploy command: 
    firebase deploy
  4. Firebase will provide you with your live Hosting URL.
